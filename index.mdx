## Build against one API

Use one bearer-authenticated API for:

- listing available models
- creating chat completions
- sending text or image input in one request

## What you can do today

<AccordionGroup>
  <Accordion icon="key" title="Authenticate with a bearer API key">
    Send your key in the `Authorization` header.

    ```http
    Authorization: Bearer sk-...
    ```
  </Accordion>

  <Accordion icon="message" title="Send text chat requests">
    Use `POST /v1/chat/completions` with `model` and `messages`.
  </Accordion>

  <Accordion icon="image" title="Send image input">
    Use OpenAI-compatible `image_url` content blocks inside a `user` message.

    The public API currently accepts `data:image/...` URLs only.
  </Accordion>
 </AccordionGroup>

## Current limits

- `stream: true` is not supported yet
- Remote image URLs are not supported
- PDF input is not supported in the public API
- Tool calling is not supported yet
